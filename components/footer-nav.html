Footer

Accessible footer navigation patterns with correct landmark structure and link clarity.

Design guidelines

UX guidelines

  • Group related links under visible headings (e.g. "Company", "Support") so users can scan column content at a glance.
  • Social media icons must have a visible or accessible label describing the destination (e.g. "Follow us on Instagram"), not just a bare icon.
  • Use <h2> or <h3> for column headings — never style a <p> or <span> to look like a heading, as this breaks document structure for screen reader users.

Visual design guidelines

  • Ensure link text and headings meet color contrast requirements against the footer's background — dark footer backgrounds often require lighter text than the rest of the page.
  • Provide adequate spacing between links so touch targets do not overlap, particularly on mobile viewports.
  • Make the footer visually distinct from the main content area — a background color change or border is a reliable separator.

Dev guidelines for web

  • Use a native <footer> element as a direct child of <body> — it maps implicitly to the contentinfo landmark role without needing role="contentinfo".
  • Use role="contentinfo" only once per page; adding it redundantly on inner elements creates duplicate landmarks.
  • Wrap each navigational link group in a separate <nav aria-labelledby="[heading-id]"> so landmark lists remain distinguishable — e.g. <nav aria-labelledby="footer-company"><h2 id="footer-company">Company</h2><ul>…</ul></nav>.
  • Use semantic <h2>/<h3> elements for column headings, not presentational markup, so screen reader users can navigate by heading.
  • Social icon links need aria-label describing the destination on the <a>, with aria-hidden="true" on the SVG: <a href="…" aria-label="Follow us on Instagram"><svg aria-hidden="true">…</svg></a>.
  • Do not wrap legal boilerplate or cookie toggles in a <nav> — overusing the landmark floods the landmark list and makes it harder for screen reader users to find the meaningful navigation regions.

Example code

<footer role="contentinfo">
  <nav aria-label="Footer navigation">
    <ul>
      <li><a href="/about">About</a></li>
      <li><a href="/privacy">Privacy policy</a></li>
      <li><a href="/accessibility">Accessibility statement</a></li>
      <li><a href="/contact">Contact us</a></li>
    </ul>
  </nav>
  <p>
    <small>&copy; 2025 Example Company. All rights reserved.</small>
  </p>
</footer>
