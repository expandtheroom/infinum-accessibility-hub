Stat / Metric Displays

Accessible stat and metric display patterns that convey meaning without relying on visual context alone.

Design guidelines

UX guidelines

  • Label and value must be visually proximate — the relationship between "1,200" and "Active users" must be clear from layout alone, not just reading order.
  • Always include the unit in the visible display (e.g. "1,200 users", "£48,200", "3.7%") — a bare number is ambiguous for all users.
  • Trend indicators (↑↓) must be paired with text (e.g. "Up 12%") — never rely on arrow direction or color alone to convey direction.
  • Large number size creates visual hierarchy but must not be the only signal of importance — critical metrics must also be distinguishable by position or explicit labeling.
  • Sparklines and inline charts embedded in a stat card need a text alternative — either a visible caption or visually-hidden description of the trend.

Visual design guidelines

  • Both the label and the value must meet WCAG AA contrast (4.5:1 for normal text) against their background — do not sacrifice legibility for a muted aesthetic.

Dev guidelines for web

  • Associate label and value semantically using <dl><dt>label</dt><dd>value</dd></dl> for multiple metrics, or aria-labeledby when a <dl> structure is not possible.
  • Include units inside the <dd> or aria-label so screen readers announce a complete, unambiguous value — e.g. "48,200 British pounds" not just "48,200".
  • Trend indicator icons must carry an accessible label: use aria-label="Up 12 percent" on the icon element, or a .visually-hidden span beside the icon with the equivalent text.
  • Real-time stat values must be wrapped in aria-live="polite" with aria-atomic="true" on the value element so screen readers announce updates without interrupting the user.
  • Throttle live region DOM writes to no more than once per second — updating more frequently on a rapid data feed will render the screen reader unusable.
  • Do not use aria-live="assertive" for routine stat updates; reserve it only for genuinely urgent, time-sensitive alerts.

Example code

<!-- Description list for key-value metric pairs -->
<dl>
  <div>
    <dt>Monthly active users</dt>
    <dd>
      <strong>24,310</strong>
      <span aria-label="up 12% from last month">&#8593; 12%</span>
    </dd>
  </div>
  <div>
    <dt>Accessibility issues resolved</dt>
    <dd><strong>143</strong></dd>
  </div>
</dl>
Monthly active users
24,310 ↑ 12%
Issues resolved
143
