Steppers

Accessible numeric stepper patterns with keyboard support and clear value announcements.

Design guidelines

UX guidelines

  • The label must clearly describe what is being incremented — "Quantity", "Number of guests", "Nights". Never leave users to infer the unit from context alone.
  • Display the unit adjacent to the value when relevant — e.g. "2 items", "3 nights", "$50". This reduces cognitive load, especially when multiple steppers appear on the same page (e.g. a booking form).
  • Annotate button accessible names in design files — specify that "−" must be labeled "Decrease [what]" and "+" must be labeled "Increase [what]", not just the symbol, so developers know to implement descriptive aria-label values.
  • Steppers are best suited to small numeric ranges (e.g. 1–10). For large ranges or precise numeric entry, provide a text input instead of, or alongside, stepper buttons. Requiring many button activations to reach a value is slow and fatiguing.

Visual design guidelines

  • Decrement and increment buttons must be clearly identifiable — use "−" and "+" symbols (or icons) at a minimum size of 44×44px touch target. Do not make them so small that motor-impaired users struggle to activate the correct button. Place decrement on the left and increment on the right, consistently throughout the product.
  • The current value must always be visible in the input — center it between the two buttons. The value display should update immediately on button activation. Never rely only on screen reader announcements; the visual display must update in real time.
  • Communicate min/max boundaries visually — when the value reaches the minimum, disable the decrement button and visually differentiate it (reduced opacity, not just color alone). Do the same for the increment button at maximum. Add a brief hint below the input if the boundaries are not obvious: "Minimum: 1".

Dev guidelines for web

  • Use <input type="number"> with min, max, and step attributes. The native number input provides Arrow Up/Arrow Down increment/decrement, browser validation, and screen reader announcement of the current value — all without additional JavaScript.
  • Decrement and increment controls must be <button type="button"> elements with descriptive aria-label values: e.g. aria-label="Decrease quantity" and aria-label="Increase quantity". Never use bare "−" and "+" as the only accessible name — screen reader users hear these as "minus" and "plus" with no context about what is changing.
  • Disable buttons at boundaries using the disabled attribute when the value is at its minimum (for decrement) or maximum (for increment). If you need the button to remain focusable (e.g. to show a tooltip), use aria-disabled="true" and intercept click and keyboard events to prevent activation.
  • Announce value changes — with a native <input type="number">, screen readers announce the value when the input is focused and when its value updates while focused. For display-only value elements (e.g. a quantity badge in a cart), use aria-live="polite" on the value container to announce changes.
  • Validate typed values on blur — if a user types a number outside the min/max range, display an inline error message linked via aria-describedby and set aria-invalid="true" on the input. Do not silently clamp the value without informing the user.
  • Link the label to the input via for/id — not to the button group as a whole. The buttons communicate their purpose through aria-label; the input communicates its purpose through its associated <label>.
  • Keyboard interaction on the input: Arrow Up increments by one step, Arrow Down decrements by one step (native behavior of type="number"). Users can also type a value directly; validate on blur.

Example code

<div>
  <span id="quantity-label">Quantity</span>
  <div role="group" aria-labelledby="quantity-label">
    <button
      type="button"
      aria-label="Decrease quantity"
      onclick="var i=document.getElementById('qty');
               if(i.value>1) i.value=parseInt(i.value)-1;"
    >
      &#x2212;
    </button>
    <input
      type="number"
      id="qty"
      name="quantity"
      value="1"
      min="1"
      max="99"
      aria-labelledby="quantity-label"
    />
    <button
      type="button"
      aria-label="Increase quantity"
      onclick="var i=document.getElementById('qty');
               if(i.value<99) i.value=parseInt(i.value)+1;"
    >
      +
    </button>
  </div>
</div>
