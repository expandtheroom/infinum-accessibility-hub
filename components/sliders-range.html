Sliders & Range Inputs

Accessible slider and range input patterns with keyboard support and value announcements.

Design guidelines

UX guidelines

  • The current value must always be displayed visually — not just in a tooltip on the thumb that appears on hover. Show the value in persistent text adjacent to the slider (above, below, or beside the track) so users with screen magnification who cannot see the full slider simultaneously can still read it.
  • Label the minimum and maximum at the ends of the track — e.g. "0" and "100", or "Low" and "High". This helps all users understand the scale without needing to drag to discover the boundaries.
  • Avoid fine-grained sliders for precise inputs — a slider with hundreds of steps is difficult to control precisely with a pointer or keyboard. Pair it with a number input field that allows direct text entry of the value.
  • For dual-thumb (range) sliders, design clearly distinct labels or value displays for both the lower and upper bounds — e.g. "Min: £50" and "Max: £200" — so the selected range is unambiguous at a glance.

Visual design guidelines

  • Indicate step increments visually when the number of steps is small — for example, a 1–5 rating slider can show tick marks at each step. For fine-grained or large-range sliders (e.g. 0–1000), tick marks are impractical; consider offering a number input as a companion control instead.
  • The slider thumb must be large enough to interact with — a minimum of 24×24px is recommended for pointer devices; on touch interfaces, ensure the touch target is at least 44×44px. The thumb must have sufficient contrast against the track background.
  • Track fill must clearly show the selected range — use a distinct color for the filled portion versus the unfilled portion. Do not rely on color alone; ensure the fill has sufficient contrast (3:1 minimum against the unfilled track).

Dev guidelines for web

  • Use <input type="range"> with an associated <label for="...">. The native element is keyboard operable and announces the current value to screen readers. Set min, max, and step attributes to define the valid range and increment size.
  • Display the current value in a <output> element linked to the input via the for attribute: <output for="volume-slider">50</output>. Update it via JavaScript on the input event. The native input does not render its value visually — this must be added explicitly.
  • For custom sliders, apply role="slider" to the draggable thumb element with these mandatory ARIA attributes: aria-valuenow (current value, updated dynamically), aria-valuemin, aria-valuemax, and aria-valuetext for non-numeric or human-readable values (e.g. aria-valuetext="Low"). Add tabindex="0" if the thumb is not a natively focusable element.
  • Implement full keyboard support on custom sliders: Arrow Right/Arrow Up increase by one step; Arrow Left/Arrow Down decrease by one step; Page Up increases by a larger step (typically 10% of range); Page Down decreases by a larger step; Home sets minimum; End sets maximum.
  • For dual-thumb range sliders, each thumb is a separate role="slider" element with its own label — e.g. aria-label="Minimum price" and aria-label="Maximum price". Prevent the lower thumb exceeding the upper and vice versa, both in interaction and in ARIA attribute values.
  • Link hint text about the range and step via aria-describedby so screen reader users hear it on focus — e.g. "Range: 0 to 100. Step: 1." This supplements the programmatic min/max/step attributes with a human-readable description.
  • Update aria-valuenow and aria-valuetext dynamically as the user drags or presses keys. Screen readers announce the new value as it changes, giving real-time feedback equivalent to what sighted users see.

Example code

<div>
  <label for="volume-slider">
    Volume:
    <output id="volume-output">50</output>%
  </label>
  <input
    type="range"
    id="volume-slider"
    name="volume"
    min="0"
    max="100"
    value="50"
    step="1"
    aria-describedby="volume-hint"
    oninput="document.getElementById('volume-output').textContent = this.value"
  />
  <p id="volume-hint">
    Use arrow keys to adjust. Range: 0&#x2013;100%.
  </p>
</div>

Use arrow keys to adjust. Range: 0–100%.
