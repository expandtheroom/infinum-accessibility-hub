Dividers & Separators

Accessible divider and separator patterns using correct semantic HTML.

Design guidelines

UX guidelines

  • Never use a divider where a heading would communicate structure better; a heading creates navigable structure, a divider does not.
  • Use whitespace as the primary separator between sections — reserve visible dividers for meaningful content boundaries only.
  • Do not place a divider between every list item; reserve them for breaks between distinct groups or sections of content.

Visual design guidelines

  • Decorative dividers should be thin and low-contrast — a subtle visual cue, not a dominant element that draws the eye away from content.
  • Maintain consistent spacing above and below every divider so the visual rhythm is predictable across the page.
  • Vertical separators in toolbars or navigation bars must have sufficient contrast against the surrounding background to be perceivable by low-vision users.

Dev guidelines for web

  • Decorative <hr> elements must have role="presentation" or aria-hidden="true" to prevent screen readers from announcing a meaningless "separator" interruption.
  • Semantic separators inside menus, listboxes, or toolbars use role="separator" on a <li> — this signals a group boundary to assistive technologies without being a focusable or selectable item.
  • Vertical separators must carry aria-orientation="vertical" so assistive technologies convey the correct orientation; the default is horizontal.
  • Never place a native <hr> directly inside <ul> or <ol> — use <li role="separator"> instead to maintain valid list structure.
  • Do not rely on dividers to create content structure that should come from headings (<h2>–<h6>) and ARIA landmark regions (<section>, <nav>, <aside>).
  • CSS-generated decorative lines via ::before / ::after are automatically ignored by screen readers and require no additional ARIA treatment.

Example code

<!-- Semantic separator between content sections -->
<section aria-labelledby="section-a">
  <h2 id="section-a">Published articles</h2>
  <p>Content here.</p>
</section>

<hr />

<section aria-labelledby="section-b">
  <h2 id="section-b">Draft articles</h2>
  <p>Content here.</p>
</section>

<!-- Decorative divider: hide from assistive technology -->
<div aria-hidden="true" class="decorative-rule"></div>

Published articles

3 articles are live and visible to readers.

Draft articles

2 drafts are saved but not yet published.

↑ Decorative divider above is aria-hidden="true"
