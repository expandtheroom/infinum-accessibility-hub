Stepper / Progress Steps

Accessible stepper and progress step patterns for multi-step processes.

Design guidelines

UX guidelines

  • Step labels should be descriptive (e.g. "Shipping", "Payment") — numbers alone ("Step 1") give users no information about what the step involves.
  • On mobile, truncate or wrap step labels gracefully — do not let long labels overlap the connector line or adjacent step circles.

Visual design guidelines

  • The current step must be visually distinct using multiple cues — a highlighted number, bold label, and a color change together, not color alone.
  • Completed steps must be marked with a check icon and a text indicator (e.g. "Completed") — a green tick without supplementary text fails WCAG 1.4.1 (Use of Color).
  • Upcoming steps should appear muted (reduced opacity or lighter text) to visually communicate they are not yet active.
  • Include a connector line between steps to reinforce the sequential relationship.

Dev guidelines for web

  • Use <nav aria-label="Checkout progress"> wrapping an <ol> — the ordered list conveys step sequence and total count to screen readers.
  • Apply aria-current="step" to the active step's <li> or its inner element — "step" is a valid aria-current token more precise than "true".
  • Communicate completed and upcoming states via text or aria-label — not CSS class alone: <a href="/checkout/shipping" aria-label="Step 1: Shipping – completed">Shipping</a>.
  • Mark connector lines and step-circle decorations with aria-hidden="true" so they are not announced by screen readers.
  • If steps are navigable (non-linear flow), use <a> elements; if not navigable (linear flow), use <li> with plain text or <span> — never use a link for a step the user cannot actually visit.
  • When content updates in-place on step navigation, move focus to the new step's heading or first field and use a live region to announce the context change: aria-live="polite".

Example code

<nav aria-label="Checkout progress">
  <ol>
    <li aria-current="step">
      <span aria-hidden="true">1</span>
      Cart
    </li>
    <li>
      <span aria-hidden="true">2</span>
      Shipping
    </li>
    <li>
      <span aria-hidden="true">3</span>
      Payment
    </li>
    <li>
      <span aria-hidden="true">4</span>
      Confirm
    </li>
  </ol>
</nav>
  1. 1
    Cart
  2. 2
    Shipping
  3. 3
    Payment
  4. 4
    Confirm
