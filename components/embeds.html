Embeds (Maps, iFrames)

Accessible embedded content patterns including iframes, maps, and third-party widgets.

Design guidelines

UX guidelines

  • Always provide a non-embed alternative. Maps should be accompanied by a plain-text address and a visible link to an external map application. The alternative must be adjacent to the embed — not buried in a footer or tooltip.
  • Use a click-to-load pattern for performance. Replace the default embed markup with a <button> that inserts the iframe on interaction. This improves page load time, reduces tracking, and gives users an explicit control point.
  • Keep iframes within the viewport height. An iframe taller than the viewport forces users to scroll inside it, which is disorienting — especially for keyboard users who may become trapped inside.
  • Include a visible caption or title below the embed. A short sentence explaining the embed's purpose helps all users — particularly those on slow connections who may see the iframe before it fully renders.

Visual design guidelines

  • Show a visible loading state while the embed loads. Display a spinner or placeholder so users know something is happening and the page has not stalled.

Dev guidelines for web

  • Every <iframe> must have a descriptive title attribute. Screen readers announce the title when focus moves into the frame. Use a specific description — for example, title="Google Map showing our London office location" — not a generic title="Map".
  • Use the sandbox attribute where appropriate. Restricting iframe permissions with sandbox reduces the attack surface and prevents the embed from interfering with the parent page's focus management.
  • Ensure there is no keyboard trap. Users must be able to Tab out of the embed using Tab, Shift+Tab, or Escape. Test every third-party embed with keyboard-only navigation — embedded maps are frequent offenders.
  • For maps, provide an <a href> link to an external map app as the accessible alternative. The link must be in the DOM adjacent to the iframe, not inside it, so users who cannot interact with the map can still get directions.
  • Lazy-load iframes with a <button> trigger, not a <div>. A button is focusable and announced to screen readers. On activation, insert the <iframe> into the DOM and move focus to it programmatically.
  • Add loading="lazy" to <iframe> elements that are below the fold. This defers iframe loading until the user scrolls near them, improving initial page performance without sacrificing content availability.

Example code

<!-- Video embed with accessible title -->
<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  title="Introduction to Web Accessibility (10 min)"
  width="560"
  height="315"
  allowfullscreen
  loading="lazy"
></iframe>

<!-- Fallback link for users who cannot access the embed -->
<p>
  <a
    href="https://www.youtube.com/watch?v=VIDEO_ID"
    target="_blank"
    rel="noopener"
  >
    Watch on YouTube: Introduction to Web Accessibility
    <span class="visually-hidden">(opens in new tab)</span>
  </a>
</p>
