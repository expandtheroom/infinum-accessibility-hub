Breadcrumbs

Accessible breadcrumb navigation patterns that communicate location within the site hierarchy.

Design guidelines

UX guidelines

  • Position breadcrumbs consistently — below the site header and above the page title — so users always know where to look.
  • The current page should appear as plain text or a visually distinct link at the end of the trail — never as an identical-looking link to the preceding items.
  • Truncate long breadcrumb paths on mobile — show the first and last items and collapse the middle with an ellipsis rather than wrapping across multiple lines.
  • Never use breadcrumbs as the primary navigation mechanism; they supplement the main nav and help users understand hierarchy.

Visual design guidelines

  • Ensure link text and separator color both meet contrast requirements against the background in all states.

Dev guidelines for web

  • Separator characters (/, ›, >) are purely decorative and must be hidden from assistive technologies; render them via CSS ::after pseudo-elements, not as text content.
  • Wrap the trail in <nav aria-label="Breadcrumb"> to create a distinct, named landmark separate from primary navigation.
  • Use an <ol> (ordered list) inside the <nav> — the ordered list communicates the hierarchical sequence to assistive technologies.
  • Each ancestor page is a <li><a href="…">; the final (current) item has aria-current="page" on its <a>, or is a <span aria-current="page"> if rendered as plain text.
  • Inject separators via CSS li + li::before { content: " / "; } — CSS-generated content is not exposed to the accessibility tree in most browsers, requiring no additional ARIA.
  • If separators must be in HTML, hide them: <span aria-hidden="true">/</span>.
  • Never place raw Unicode chevrons or arrows between items without aria-hidden="true" — screen readers may announce them as "greater-than sign".

Example code

<nav aria-label="Breadcrumb">
  <ol>
    <li><a href="/">Home</a></li>
    <li><a href="/components">Components</a></li>
    <li>
      <a href="/components/navigation" aria-current="page">
        Navigation
      </a>
    </li>
  </ol>
</nav>
  1. Home
  2. /
  3. Components
  4. /
  5. Navigation
