Top Navigation Bar / Header

Accessible top navigation bar patterns with keyboard navigation and correct landmark structure.

Design guidelines

UX guidelines

  • The logo or brand link must be clearly visible and distinct — users expect it to navigate to the home page, so it should appear at the top-left on LTR layouts.
  • At mobile breakpoints, replace the nav with an accessible hamburger menu pattern — the collapsed state must be equally navigable to the expanded state.
  • If the header is sticky, account for its height in the page scroll offset so focused elements below are never obscured by the header.

Visual design guidelines

  • Primary nav items must be spaced comfortably for both click and touch — insufficient spacing causes mis-taps on mobile.
  • The active page link must be visually indicated using more than color alone — bold text, an underline, or a border beneath the active link ensures color-blind users can identify the current page.
  • All interactive nav items must have visible hover and focus states designed as part of the component — do not rely solely on the browser default focus ring.
  • Ensure sufficient contrast between the nav background color and link text in all states (default, hover, active, focus).

Dev guidelines for web

  • Use a native <header> as a direct child of <body> — it maps implicitly to the banner landmark role without needing role="banner".
  • Place the primary nav inside <nav aria-label="Primary navigation"> — each <nav> on the page must have a distinct aria-label so screen reader landmark lists are unambiguous.
  • Apply aria-current="page" to the active link in the nav — screen readers announce this as "current page", giving non-sighted users the same orientation cue as the visual active state.
  • The hamburger toggle must be <button aria-expanded="false" aria-controls="mobile-nav" aria-label="Open menu">; update aria-expanded to "true" when the mobile nav is open.
  • For sticky headers, set scroll-margin-top on all anchor targets equal to the header height: :root { --header-height: 80px; } :target, [id] { scroll-margin-top: var(--header-height); }.
  • The skip link must point past the header: <a class="skip-link" href="#main-content">Skip to main content</a> — see the Skip to Content Link page for full implementation guidance.

Example code

<header role="banner">
  <a href="/" aria-label="Acme Corp &#x2013; Home">
    Acme Corp
  </a>

  <!-- Mobile menu toggle -->
  <button
    type="button"
    aria-haspopup="true"
    aria-expanded="false"
    aria-controls="main-nav"
    aria-label="Open navigation menu"
  >
    <span aria-hidden="true">&#9776;</span>
  </button>

  <nav id="main-nav" aria-label="Main navigation">
    <ul>
      <li><a href="/" aria-current="page">Home</a></li>
      <li><a href="/products">Products</a></li>
      <li><a href="/about">About</a></li>
    </ul>
  </nav>
</header>
