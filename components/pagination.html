Pagination

Accessible pagination patterns that clearly communicate current position and allow keyboard navigation.

Design guidelines

UX guidelines

  • Label previous and next controls with visible text, not just arrow glyphs — use "Previous" and "Next" (with an arrow as a supplementary icon if desired).
  • Use an ellipsis (…) to indicate a truncated range of pages — annotate it with an accessible label such as "More pages" for screen reader users.
  • At the first page, the Previous control should appear with reduced opacity and a not-allowed cursor, or be removed entirely — never left as an active-looking link.
  • At the last page, apply the same treatment to the Next control.

Visual design guidelines

  • The current page must be visually distinct using more than color alone — combine a bold numeral, an underline, or a border indicator with any background color change.
  • Page number links must have adequate touch target size (minimum 44×44px recommended) so they are easy to tap on mobile.

Dev guidelines for web

  • Wrap the component in <nav aria-label="Pagination"> containing a <ul> with <li><a> items — pagination links navigate to new pages, so <a> is correct.
  • Add aria-current="page" to the current page link: <a href="/results?page=3" aria-current="page">3</a>.
  • Previous and Next must be <a> elements (not <button>) with descriptive aria-label values: aria-label="Previous page" and aria-label="Next page"; hide arrow glyphs with aria-hidden="true".
  • Give numbered page links explicit aria-label values such as "Page 1", "Page 2" so screen reader users can distinguish them in a links list.
  • Do not use the disabled attribute on links — it has no semantic effect on <a> elements. Instead, omit the Previous link on page 1 and the Next link on the last page, or replace them with non-interactive, non-focusable elements.
  • If using a "Load more" pattern instead of pagination, place a <button type="button">Load more results</button> and announce newly loaded content via a live region: <div aria-live="polite" aria-atomic="true">Showing 40 of 200 results</div>.

Example code

<nav aria-label="Search results pagination">
  <ul role="list">
    <li>
      <a href="/results?page=1" aria-label="Previous page">
        <span aria-hidden="true">&#8249;</span>
      </a>
    </li>
    <li><a href="/results?page=1">1</a></li>
    <li>
      <a href="/results?page=2" aria-current="page">2</a>
    </li>
    <li><a href="/results?page=3">3</a></li>
    <li>
      <a href="/results?page=3" aria-label="Next page">
        <span aria-hidden="true">&#8250;</span>
      </a>
    </li>
  </ul>
</nav>
