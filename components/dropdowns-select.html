Dropdowns & Select Menus

Accessible dropdown and select menu patterns, from native HTML selects to custom implementations.

Design guidelines

UX guidelines

  • The selected value must always be visible in the control — once a user makes a selection it must remain displayed in the trigger element at all times, not just in the open list. Never hide the selected value behind a placeholder that reappears after selection.
  • Avoid option lists with more than 10–15 items without type-ahead or filtering. Long lists with no way to jump to an item are slow and fatiguing for all users. If the list is very long, design a combobox (filterable dropdown) rather than a plain select.
  • A placeholder option must be visually differentiated from real options — use lighter text or italic styling, and never give it a selectable value. Label it something like "Select an option" so users understand the field has not been completed.
  • Annotate custom dropdowns in design files with the expected ARIA pattern (combobox, listbox, option roles) so developers have the semantics specified before implementation begins.

Visual design guidelines

  • All option text must meet 4.5:1 contrast against the option list background. Option groups (e.g. continent names above country lists) must be visually distinct — use a different weight, size, or color — but must still meet contrast requirements.
  • Disabled options must look non-interactive — use reduced opacity or a muted color. Annotate in design files that disabled options must also carry aria-disabled="true" in implementation so they are not interactive for keyboard users.
  • The dropdown indicator (chevron icon) should be decorative. Do not rely on the chevron alone to convey that the control is a dropdown — ensure the control looks interactive (bordered, with sufficient background contrast) in its default state.

Dev guidelines for web

  • Prefer the native <select> element paired with an associated <label for="...">. The native element is keyboard operable, screen-reader compatible, and touch-friendly across all platforms at zero additional cost. Use <optgroup label="..."> for grouped options.
  • For custom dropdowns, implement the ARIA combobox pattern: role="combobox" on the trigger input, role="listbox" on the option container, and role="option" on each item. Set aria-expanded="true/false" on the combobox to reflect open/closed state, and aria-haspopup="listbox".
  • Use aria-activedescendant on the combobox input to track the focused option without moving DOM focus away from the input. Set its value to the id of the currently highlighted option as the user navigates. Set aria-selected="true" on the chosen option and aria-selected="false" on all others.
  • Implement full keyboard support in custom dropdowns: Arrow Down/Arrow Up navigate options, Enter selects and closes, Escape closes without selecting and returns focus to the trigger, Home/End jump to first/last option.
  • Type-ahead filtering is mandatory for custom dropdowns — it is provided automatically by native <select>. Typing a letter should move focus to the first option beginning with that character; rapid typing of multiple characters should narrow the match.
  • Set aria-disabled="true" on individual role="option" elements that are unavailable, and prevent keyboard selection of those options. For native <select>, use the disabled attribute on <option> elements.
  • When the dropdown closes after selection, return focus to the trigger element. Never leave focus in an undefined location or allow it to fall to the top of the page.
  • The dropdown indicator (chevron icon) should be decorative and marked aria-hidden="true" in implementation.

Example code

<!-- Native select &#x2014; always prefer this -->
<div>
  <label for="country-select">Country</label>
  <select id="country-select" name="country">
    <option value="">Select a country</option>
    <option value="au">Australia</option>
    <option value="ca">Canada</option>
    <option value="us">United States</option>
  </select>
</div>

<!-- Grouped options -->
<div>
  <label for="timezone-select">Timezone</label>
  <select id="timezone-select" name="timezone">
    <optgroup label="Americas">
      <option value="America/New_York">Eastern (UTC&#x2212;5)</option>
      <option value="America/Los_Angeles">Pacific (UTC&#x2212;8)</option>
    </optgroup>
    <optgroup label="Europe">
      <option value="Europe/London">London (UTC+0)</option>
      <option value="Europe/Zagreb">Zagreb (UTC+1)</option>
    </optgroup>
  </select>
</div>
