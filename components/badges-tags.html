Badges & Tags / Chips

Accessible badge, tag, and chip patterns with clear labeling and correct interaction semantics.

Design guidelines

UX guidelines

  • Count badges (e.g. notification bubbles) must be positioned so they do not obscure the icon or label they annotate.
  • Removable chips must display the remove control persistently, not only on hover, to support keyboard-only users who cannot hover.

Visual design guidelines

  • Status badges must convey meaning through color + icon + text — never color alone. A green "Active" and red "Error" badge are indistinguishable to color-blind users.
  • Badge contrast must meet at least 3:1 against its surrounding background to satisfy WCAG non-text contrast requirements.
  • Chip remove (×) buttons must have a touch target of at least 24 × 24 px and be visually distinct from the chip label at all times.
  • Interactive chips must have clearly visible hover and focus states — do not rely solely on background color changes that may be invisible to color-blind users.
  • Avoid animated badge transitions (e.g. count incrementing with motion) unless prefers-reduced-motion is respected.

Dev guidelines for web

  • Decorative badges (e.g. a colored dot duplicating visible text) must have aria-hidden="true" to suppress redundant screen reader announcements.
  • Count badges must expose their value to assistive technology — either via the parent's aria-label (e.g. aria-label="Notifications, 3 unread") or a span inside the badge.
  • Interactive chips that toggle a filter should be <button aria-pressed="true/false"> or <input type="checkbox"> wrapped in a <label>. Use <a href> only when the chip navigates.
  • Removable chips: place a <button aria-label="Remove [chip label]"> inside the chip element — never use a bare × character without an accessible name.
  • After a chip is removed, move focus programmatically to the next chip, the previous chip, or the associated input field so keyboard users are not left in a void.
  • Status badges paired with visible text do not need an additional aria-label; when a badge's color icon is the only indicator, provide aria-label or a .visually-hidden span with the status text.
  • Dynamic count updates: wrap the value in <span aria-live="polite" aria-atomic="true"> so screen readers announce changes without interrupting the user.

Example code

<!-- Informational badge -->
<span class="badge" aria-label="12 unread notifications">12</span>

<!-- Status badge (color-independent) -->
<span class="badge badge--success">
  <span aria-hidden="true">&#10003;</span> Published
</span>

<!-- Removable tag / chip -->
<span class="tag">
  Accessibility
  <button
    type="button"
    aria-label="Remove tag: Accessibility"
  >
    <span aria-hidden="true">&times;</span>
  </button>
</span>
🔔 12
✓ Published Accessibility
