Tooltips

Accessible tooltip patterns that are keyboard-triggerable and never relied upon for essential information.

Design guidelines

UX guidelines

  • Tooltips must appear on both hover and keyboard focus — keyboard users who cannot use a mouse must be able to access the same supplementary information.
  • Keep tooltip text short — maximum 1–2 lines. Tooltips are for supplementary clarification, not detailed explanations.
  • Never place interactive content (links, buttons, form controls) inside a tooltip. Tooltips cannot receive focus, so any interactive elements inside are completely inaccessible to keyboard users. Use a popover pattern instead.
  • Apply a 300–500 ms delay before showing the tooltip on hover to prevent it appearing disruptively as the user's pointer passes over the trigger. No delay is needed on keyboard focus.
  • Never use tooltips for essential information. If a user needs the information to complete a task, it must be permanently visible — not hidden behind a hover.

Visual design guidelines

  • Position tooltips to avoid viewport clipping — flip to the opposite side of the trigger if the tooltip would extend beyond the screen edge.
  • Tooltip text and background must meet sufficient color contrast — WCAG requires 3:1 for UI component boundaries, but 4.5:1 for the text itself against its background is best practice.

Dev guidelines for web

  • Give the tooltip container role="tooltip" and a unique id. The trigger element references it via aria-describedby: <button aria-describedby="tip-save">Save</button><span role="tooltip" id="tip-save">Saves your changes</span>.
  • aria-describedby reads the referenced element regardless of visual visibility — tooltip text is available to screen reader users even when the bubble is not shown on screen.
  • Show the tooltip on both :hover and :focus-visible using CSS, or toggle visibility on mouseenter/mouseleave and focus/blur events in JavaScript.
  • Pressing Escape must dismiss the tooltip without moving focus (WCAG 1.4.13 — Content on Hover or Focus). The tooltip must also remain visible when the pointer moves over the tooltip bubble itself.
  • Hide the tooltip with the hidden attribute or visibility:hidden — not display:none. The tooltip element must exist in the DOM at all times for aria-describedby to work correctly.
  • Never put links or buttons inside a tooltip. If interactive content is needed in a triggered overlay, use a popover with aria-expanded on the trigger and proper focus management.

Example code

<button
  type="button"
  aria-describedby="tooltip-copy"
  id="copy-btn"
>
  <span aria-hidden="true">&#128203;</span>
  <span class="visually-hidden">Copy to clipboard</span>
</button>

<div role="tooltip" id="tooltip-copy">
  Copy to clipboard
</div>
Copy to clipboard

Hover or focus the button to reveal the tooltip. It uses role="tooltip" + aria-describedby.
