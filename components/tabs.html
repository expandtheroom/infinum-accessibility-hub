Tabs

Accessible tab interface patterns using the correct ARIA roles, states, and keyboard interactions.

Design guidelines

UX guidelines

  • Tab panel content must be clearly associated below the tab strip — avoid designs where the panel appears detached or ambiguous.
  • Avoid wrapping tabs across multiple lines where possible; if overflow occurs, use horizontal scroll or a dropdown variant rather than wrapping.

Visual design guidelines

  • The active tab must be visually distinct from inactive tabs using more than color alone — combine bold text with a bottom border or background indicator.
  • The selected indicator (e.g. border-bottom) must have a contrast ratio of at least 3:1 against adjacent colors to satisfy WCAG 1.4.11 (Non-text Contrast).
  • Inactive tabs must remain legible — do not use such low contrast on inactive labels that they appear disabled when they are still selectable.
  • The focus ring on a tab must be visible and contained within the tab strip, not clipped by overflow or container edges.

Dev guidelines for web

  • Apply role="tablist" to the tab strip container, role="tab" to each tab button, and role="tabpanel" to each content panel.
  • Set aria-selected="true" on the active tab and aria-selected="false" on all others — do not omit aria-selected on inactive tabs, as an absent attribute is not equivalent to "false" in all browsers.
  • Link each tab and panel bidirectionally: tab has aria-controls="panel-id"; panel has aria-labelledby="tab-id".
  • Use a roving tabindex: tabindex="0" on the selected tab, tabindex="-1" on all others. Arrow keys (←/→, or ↑/↓ for vertical) move focus between tabs; Tab moves focus into the active panel.
  • Hide inactive panels with display: none or the hidden attribute — not visibility: hidden or opacity: 0, which leave panel content reachable by Tab.
  • Choose either manual activation (Enter/Space selects a focused tab) or automatic activation (arrow key focus immediately selects) — be consistent across all tab instances on the site.

Example code

<div>
  <div role="tablist" aria-label="Account settings">
    <button
      role="tab"
      id="tab-profile"
      aria-selected="true"
      aria-controls="panel-profile"
    >
      Profile
    </button>
    <button
      role="tab"
      id="tab-security"
      aria-selected="false"
      aria-controls="panel-security"
      tabindex="-1"
    >
      Security
    </button>
  </div>
  <div
    role="tabpanel"
    id="panel-profile"
    aria-labelledby="tab-profile"
  >
    <h3>Profile settings</h3>
    <p>Update your name, bio, and avatar here.</p>
  </div>
  <div
    role="tabpanel"
    id="panel-security"
    aria-labelledby="tab-security"
    hidden
  >
    <h3>Security settings</h3>
    <p>Manage your password and two-factor authentication.</p>
  </div>
</div>
