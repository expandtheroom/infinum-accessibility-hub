Sticky / Fixed Elements

Accessible patterns for sticky and fixed position elements that avoid obstructing content or focus.

Design guidelines

UX guidelines

  • Keep sticky header height as small as possible. Every pixel of persistent height reduces the usable viewport for content. Strip the header to the minimum required: logo, primary navigation, and any critical actions.
  • Design with scroll offset in mind. The sticky element will overlap anchor link targets and keyboard-focused elements that scroll into its area. Spec the header height clearly so developers can apply the correct scroll-margin-top or scroll-padding-top values.
  • FABs must not obscure important content. Audit every page at mobile viewport widths to confirm the FAB does not permanently sit over primary actions, form fields, or the last item in a list.
  • Sticky sidebars should scroll independently of the main content. Use overflow-y: auto on the sidebar so its content remains reachable without affecting the main scroll context.
  • Question whether sticky behavior serves users. Sticky elements add visual complexity and accessibility risk. Only make something sticky if removing it would genuinely harm the user's ability to navigate or complete a task.

Visual design guidelines

  • Floating action buttons (FABs) must have clear visible labels, not just icons. Icon-only FABs require guesswork from all users and a tooltip or screen reader label for those who cannot see them. Pair the icon with visible text wherever space allows.

Dev guidelines for web

  • Set scroll-padding-top: [header-height] on the html element. This single rule corrects scroll position globally for both anchor links and keyboard-focused elements that scroll into view, compensating for the sticky header across the entire page.
  • Alternatively, apply scroll-margin-top: [header-height] to individual anchor link targets. This is useful when different sections of the page need different offsets, or when scroll-padding-top cannot be set globally.
  • Update the offset value at each responsive breakpoint where header height changes. Use a CSS custom property — for example --header-height: 80px — and reference it in both the sticky positioning and the scroll offset so both values stay in sync.
  • FABs must be <button> elements with an aria-label. Do not use a <div> or <a> for a button-like action. The aria-label provides the accessible name for screen reader users who cannot see the icon.
  • position: fixed elements must not cause reflow that shifts other content. Fixed elements are removed from document flow by default, but any JavaScript-driven layout changes triggered by them (such as adjusting body padding to account for a fixed banner) must be tested to ensure they do not introduce unexpected shifts.
  • Test keyboard navigation at every breakpoint. Tab through every interactive element on the page and confirm that the focus indicator is fully visible — never clipped or hidden behind a sticky element — at each viewport width.

Example code

<!-- Sticky table header -->
<div style="max-height: 200px; overflow-y: auto;">
  <table>
    <thead>
      <tr>
        <th scope="col" style="position: sticky; top: 0;">Name</th>
        <th scope="col" style="position: sticky; top: 0;">Role</th>
        <th scope="col" style="position: sticky; top: 0;">Status</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>Alice</td><td>Designer</td><td>Active</td>
      </tr>
      <!-- more rows &#x2026; -->
    </tbody>
  </table>
</div>
