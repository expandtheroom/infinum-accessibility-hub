Images & Icons

Accessible image and icon patterns covering alt text, decorative images, and SVG accessibility.

Design guidelines

UX guidelines

  • Informative images need alt text that conveys the same purpose as the image. Describe the meaning, not the appearance. A chart that shows a 40% increase in sign-ups should be described by that insight, not labeled "a bar chart".
  • Decorative images need no alt — but the attribute must still be present. Set alt="" on purely aesthetic images (background textures, illustrative stock photography). Omitting the attribute entirely causes screen readers to fall back to reading the filename.
  • Functional images (logos as links) need alt text describing the function. When a logo is the only content inside a link, the alt text should describe the destination — for example, alt="Accessibility Hub – Home" — not just the logo's appearance.
  • Complex images (charts, diagrams) need both a short alt and a longer description. The alt attribute gives a summary; a supplementary paragraph, <details> block, or linked page provides the full description.
  • Captions are supplementary context, not a substitute for alt text. A <figcaption> provides context visible to all users; alt text is the text alternative for users who cannot see the image. Both serve different purposes and can coexist.

Visual design guidelines

  • Image containers must not crop important content. Ensure that image aspect ratios and container constraints are tested across viewport widths so faces, data labels, and other critical content are never clipped.

Dev guidelines for web

  • Use <img alt="descriptive text"> for informative images. Write alt text that communicates the same meaning the image conveys in context — not a visual inventory of what is depicted.
  • Use alt="" for decorative images. The empty attribute is required; omitting it entirely causes screen readers to announce the filename instead.
  • Functional images inside links: describe the destination. A logo-link should use alt="Accessibility Hub – Home" so keyboard and screen reader users know where the link goes.
  • Inline SVG icons: add aria-hidden="true" to the <svg> and provide a .visually-hidden label on the parent button or link. Also set focusable="false" on the SVG to prevent it entering the tab order in older browsers.
  • Use <figure> and <figcaption> for complex images. This associates the visible caption with the image element semantically, so screen reader users understand the relationship.
  • For long descriptions, use aria-describedby pointing to an adjacent paragraph, or provide a "View full description" link. Do not rely solely on alt text to convey detailed chart or diagram data — the full description must be reachable by all users.

Example code

<!-- Informative image -->
<img
  src="bar-chart-q4.png"
  alt="Bar chart: Q4 revenue up 18% year-over-year, reaching $4.2M"
/>

<!-- Decorative image: empty alt -->
<img src="divider-wave.svg" alt="" />

<!-- Icon-only button: label the button, not the icon -->
<button type="button" aria-label="Delete item">
  <svg aria-hidden="true" focusable="false">&#8230;</svg>
</button>

<!-- Icon with text: hide the icon from AT -->
<button type="button">
  <svg aria-hidden="true" focusable="false">&#8230;</svg>
  Save changes
</button>
