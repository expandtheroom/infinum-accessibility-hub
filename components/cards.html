Cards

Accessible card patterns with correct heading structure, link semantics, and keyboard navigation.

Design guidelines

UX guidelines

  • Each card should have one primary action — the card heading link — that is visually prominent and reached first by keyboard users.
  • Secondary actions (e.g. Save, Share) must be visually smaller and clearly separated from the primary link so they are not mistaken for it.
  • Card images must have meaningful alt text when they convey content not present elsewhere; use alt="" for purely decorative images.
  • Avoid placing too many interactive elements on a single card; each additional tab stop increases navigation burden for keyboard users.

Visual design guidelines

  • Interactive cards must have a clearly visible focus ring — the ring should visually encompass the entire card or at minimum the primary link, not just a thin underline.
  • Cards must have a visible hover state (e.g. shadow, border change, background shift) so pointer users can identify the interactive affordance.

Dev guidelines for web

  • The card heading is the primary link: use <h2><a href="/…">Card title</a></h2> — never wrap the entire card in <a>, which creates an over-verbose screen reader announcement.
  • Do not duplicate link targets: if the image and heading both link to the same URL, merge them into one link with alt="" on the image so the accessible name comes from the heading text only.
  • Secondary actions must be separate <button> or <a> elements placed after the primary link in DOM order, each with a descriptive aria-label that includes the card subject (e.g. aria-label="Save: The joy of dogs").
  • Group cards in a <ul role="list"> when they form a collection so screen reader users hear the item count.
  • To make the entire card surface clickable without wrapping it in <a>, use the CSS pseudo-element trick: .card { position: relative } .card a.primary::after { content: ""; position: absolute; inset: 0 } — secondary elements need position: relative and a higher z-index to remain independently clickable.
  • Never use tabindex values greater than 0 to control focus order inside or across cards.
