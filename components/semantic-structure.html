Semantic Structure

Guidelines for using HTML landmarks, sectioning elements, and heading hierarchies so that assistive technology users can navigate and understand page structure without visual context.

Design guidelines

UX guidelines

  • Every section of content should have a visible label. Screen reader users navigate by landmark and heading. A section with no visible heading is an unnamed region — users cannot know what it contains until they read through it. Prefer a visible <h2> or <h3> over a hidden aria-label wherever possible; visible labels benefit all users, including those with cognitive disabilities.
  • When a visible heading is not practical, require an aria-label or aria-labelledby. Spec this in design annotations for any <section>, <nav>, or <aside> that lacks a heading. Examples: a secondary navigation bar, a site-wide search region, a promotional banner. Without a label, these landmarks appear unnamed in screen reader landmark lists and are difficult to distinguish.
  • Annotate heading levels in design files. Specify whether a typographic style maps to an h1, h2, h3, etc. Visual size alone does not determine level — a visually large decorative title in a hero section may still be the page's only h1, while a smaller section title underneath it is an h2. Mismatched visual hierarchy and DOM hierarchy are a common source of accessibility failures.
  • Each page must have exactly one h1. It should describe the page's primary topic or purpose. Users navigating by heading use the h1 as a landmark to confirm they have arrived at the right page.
  • Heading levels must not skip. Moving from an h2 directly to an h4 creates a gap in the outline that breaks screen reader navigation. A user jumping to "heading level 3" will miss content that was incorrectly placed at level 4. Design outlines should map to a logical hierarchy: page title (h1), major sections (h2), subsections (h3), and so on.
  • Navigation landmarks should always be labeled when multiple exist on a page. If a page has both a primary navigation and a secondary or utility navigation, each <nav> must have a distinct label so users can tell them apart in the landmark list.

Visual design guidelines

  • Do not style headings by level alone. An h2 in one context may need to look visually smaller than an h3 in another (e.g. a card title inside a section). Use CSS classes to control visual appearance independently of the heading level. Never choose a heading level for its visual output — choose it for its position in the document outline.
  • Landmark boundaries should be visually apparent to sighted users. While invisible to sighted users, landmarks like <main>, <nav>, and <aside> help all users. Use whitespace, borders, or background colors to visually reinforce the separation between distinct page regions, reducing cognitive load for everyone.

Dev guidelines for web

  • Use native sectioning elements, not generic <div>s, for landmark regions. The browser exposes these as ARIA landmarks automatically: <header role="banner">, <nav>, <main>, <aside>, <footer role="contentinfo">, <section>, <article>. A <div> is never a landmark regardless of its class name.
  • Label every <section> and <nav>. A <section> without an accessible name is not exposed as a landmark — it is treated like a generic container. Add aria-labelledby="[heading-id]" pointing to its visible heading, or aria-label="[descriptive name]" when no visible heading exists.
  • Every page must contain exactly one <main> element. Add id="main-content" and tabindex="-1" so that skip links and JavaScript navigation can move focus to it programmatically.
  • Heading levels must reflect document structure, not visual style. Use CSS classes to control font size and weight. Never jump from <h2> to <h4>; never use an <h1> for a sidebar widget title simply because it looks right.
  • Multiple <nav> elements must have distinct aria-label values. Screen readers list all navigation landmarks together. "Main navigation", "Breadcrumb", "Footer navigation", and "Pagination" are all informative labels; a second unnamed <nav> is announced identically to the first and creates confusion.
  • <article> is for self-contained, independently distributable content. Use it for blog posts, news articles, cards, and comments — content that makes sense on its own when removed from its page context. Do not use <article> as a generic layout wrapper.
  • <aside> is for tangentially related content. Use it for sidebars, pull quotes, and related-links panels whose removal would not break the primary content. Screen readers announce it as a "complementary" landmark; placing primary content inside an <aside> misleads users about its importance.
  • Use <header> and <footer> inside sectioning elements for section-level headers and footers. A <header> inside <article> is scoped to that article, not the page. Only a <header> that is a direct child of <body> (or outside all sectioning elements) maps to the banner landmark role.

Example code

<!-- Page-level landmark structure -->
<body>
  <a class="skip-link" href="#main-content">
    Skip to main content
  </a>

  <header role="banner">
    <!-- Site logo, primary nav -->
  </header>

  <nav aria-label="Main navigation">
    <!-- Primary links -->
  </nav>

  <main id="main-content" tabindex="-1">

    <!-- Each section labelled by its heading -->
    <section aria-labelledby="intro-heading">
      <h1 id="intro-heading">Page title</h1>
      <p>Introduction content.</p>
    </section>

    <section aria-labelledby="details-heading">
      <h2 id="details-heading">Details</h2>
      <p>Section content.</p>

      <section aria-labelledby="subsection-heading">
        <h3 id="subsection-heading">Subsection</h3>
        <p>Nested content.</p>
      </section>
    </section>

  </main>

  <aside aria-labelledby="related-heading">
    <h2 id="related-heading">Related articles</h2>
    <!-- Tangential content -->
  </aside>

  <footer role="contentinfo">
    <nav aria-label="Footer navigation">
      <!-- Footer links -->
    </nav>
  </footer>
</body>
<body>
<header role="banner"> // banner landmark
<nav aria-label="Main navigation">
<main id="main-content" tabindex="-1">
<section aria-labelledby="h1-id">
<h1>Page title</h1>
<section aria-labelledby="h2-id">
<h2>Major section</h2>
<h3>Subsection</h3>
</main>
<aside aria-labelledby="aside-h2"> // complementary landmark
<footer role="contentinfo">
</body>

  • 🏷 main
