Video

Accessible video patterns with captions, audio descriptions, and keyboard-accessible player controls.

Design guidelines

UX guidelines

  • Make the captions toggle button always visible in player controls. Users should be able to enable or disable captions without hunting through settings menus. Place the captions button prominently alongside play/pause.
  • Turn captions on by default. Research consistently shows that captions benefit users beyond those with hearing loss — including users in noisy environments, non-native speakers, and users with cognitive disabilities. Defaulting to on serves more users.
  • Provide an audio description toggle or an alternate described version. Visual-only information — on-screen text, demonstrations, scene changes without narration — must be communicated to users who cannot see the video.
  • Place a transcript link adjacent to the player. The transcript must be reachable without interacting with the video. A clearly labeled link immediately below the player is the expected location.
  • Never autoplay video with sound. Autoplay with audio disrupts screen reader speech and is prohibited by WCAG 1.4.2 (Audio Control) unless a pause or mute mechanism is among the first three interactive elements on the page.
  • Looping background videos — such as hero section videos — must include a visible pause button. WCAG 2.2.2 (Pause, Stop, Hide) requires a mechanism to pause or stop any content that moves, scrolls, or auto-updates for more than five seconds. The pause control must be keyboard-accessible and positioned so it is reachable without scrolling past the video.

Visual design guidelines

  • Ensure player controls are large enough to interact with. All interactive controls — play, pause, seek, volume, captions — should meet a minimum 44×44 px touch target size.
  • Caption text must have sufficient contrast over video content. White text with a black outline or a semi-transparent black box behind captions ensures legibility regardless of what is showing in the video frame.

Dev guidelines for web

  • Use <video controls> — do not hide native controls. The controls attribute provides browser-managed play, pause, seek, volume, and fullscreen that are keyboard accessible without additional ARIA work.
  • Add <track kind="captions" src="captions.vtt" srclang="en" label="English captions"> inside the <video> element. The kind="captions" value is semantically distinct from kind="subtitles" — captions include non-speech audio information; subtitles do not.
  • Enable captions by default using the default attribute on the <track> element. Without it, captions are available but off by default in most browsers.
  • Never use the autoplay attribute on videos with audio. If autoplay is required for a decorative background video, add muted and ensure prefers-reduced-motion pauses or removes the video.
  • Implement a pause/play toggle for looping background videos. Place a <button> over or directly below the video. Update its accessible name to reflect the current state — aria-label="Pause background video" when playing, aria-label="Play background video" when paused. Call video.pause() / video.play() on click. Also pause automatically when prefers-reduced-motion: reduce is active: 
    const video = document.querySelector('.hero-video');
const btn = document.querySelector('.hero-video-toggle');

// Respect reduced-motion preference immediately
if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
  video.pause();
}

btn.addEventListener('click', () => {
  if (video.paused) {
    video.play();
    btn.setAttribute('aria-label', 'Pause background video');
  } else {
    video.pause();
    btn.setAttribute('aria-label', 'Play background video');
  }
});
  • Set preload="metadata" to load only duration and track information on page load, deferring the full video download until the user presses play.
  • Provide audio descriptions via a second <track kind="descriptions"> or an alternate described video version. Both approaches satisfy WCAG 1.2.5 (Audio Description, Pre-recorded) at AA level.

Example code

<figure>
  <figcaption id="video-label">
    Tutorial: Getting started with keyboard testing (5 min)
  </figcaption>
  <video
    controls
    aria-labelledby="video-label"
    poster="thumbnail.jpg"
  >
    <source src="keyboard-testing.mp4" type="video/mp4" />
    <track
      kind="captions"
      src="captions-en.vtt"
      srclang="en"
      label="English captions"
      default
    />
    <p>
      Your browser does not support HTML video.
      <a href="keyboard-testing.mp4">Download the video</a>.
    </p>
  </video>
</figure>
