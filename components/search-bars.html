Search Bars

Accessible search input patterns including labeling, submission behavior, and results announcement.

Design guidelines

UX guidelines

  • Include a search icon inside or immediately adjacent to the input to communicate the field's purpose visually. The icon alone is not sufficient — pair it with a visible or screen-reader-accessible label. The icon itself should be decorative (aria-hidden="true") since the label carries the accessible name.
  • The submit or search button must be clearly labeled — use "Search" as visible text, or a search icon with a tooltip and accessible name. Never provide only an unlabeled icon button as the sole way to trigger a search.
  • Empty results must show a clear message — "No results for [query]" with a suggestion to refine the search. Design this message prominently; do not leave an empty container that looks like a loading failure.
  • Annotate autocomplete in design files — specify the keyboard interactions (arrow keys to navigate, Enter to select, Escape to dismiss) so developers have the expected behavior documented before implementation.

Visual design guidelines

  • A clear button (×) must be visible when the input has a value — it should appear inside the input field on the right, or immediately adjacent. Design it with a minimum 44×44px touch target and an accessible label ("Clear search"). It should disappear when the field is empty to avoid visual clutter.
  • Autocomplete suggestion lists must have sufficient contrast — option text must meet 4.5:1 against the list background. The highlighted (active) suggestion must be visually distinct from unselected options using more than color — use a background fill plus a border, or a change in text weight.
  • Design a visible loading state for async suggestions — an animated spinner or "Searching…" text within the suggestion area. The loading state must not rely on color or animation alone.

Dev guidelines for web

  • Wrap the search input in <form role="search"> — this landmark role allows screen reader users to navigate directly to the search form using landmark navigation shortcuts. Use role="search" on a <form>, not on a <div>, so the native form submission behavior (including Enter key) is preserved.
  • Provide a visible <label> or aria-label on the input — placeholder text is not a sufficient accessible name. A common accessible pattern uses a visually hidden label: <label for="site-search" class="visually-hidden">Search this site</label> paired with a visible placeholder.
  • The clear button must be a <button aria-label="Clear search"> — not a styled <span> or <div>. Show it conditionally when the input has a value and move focus back to the input after clearing.
  • For autocomplete suggestions, implement the ARIA combobox pattern: role="combobox" on the input, aria-expanded="true/false", aria-haspopup="listbox", aria-controls pointing to the suggestion list. The list uses role="listbox"; each suggestion uses role="option". Use aria-activedescendant to track the highlighted suggestion without moving DOM focus from the input.
  • Announce result counts with aria-live="polite": <p aria-live="polite" aria-atomic="true">24 results for "accessible forms"</p>. Update this element after each search. For zero results, include a suggestion: "No results for 'accsible forms'. Did you mean 'accessible forms'?"
  • Manage focus after same-page search submission — move focus to the results heading (e.g. an <h2 tabindex="-1">) or the first result so keyboard and screen reader users are oriented to the updated content.
  • For full-page search result pages, update the page <title> to include the query — e.g. "Search results for 'accessible forms' – Accessibility Hub" — so screen reader users hear the context immediately on page load.

Example code

<search>
  <form role="search" action="/search">
    <label for="site-search">Search the site</label>
    <input
      type="search"
      id="site-search"
      name="q"
      placeholder="e.g. buttons, WCAG, contrast"
      autocomplete="off"
    />
    <button type="submit">
      <span class="visually-hidden">Submit search</span>
      <span aria-hidden="true">&#128269;</span>
    </button>
  </form>
</search>
