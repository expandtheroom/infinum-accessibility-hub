Popovers

Accessible popover patterns with keyboard triggering, focus management, and dismissal behavior.

Design guidelines

UX guidelines

  • Include a visible close button inside the popover — do not rely solely on clicking outside or pressing Escape, as both require the user to know that mechanism exists.
  • If the popover contains only read-only information (no interactive elements), consider using the Tooltips pattern instead — see the Tooltips page for the distinction.
  • Popover content should be concise — for complex tasks requiring more space, consider a modal or a dedicated page instead.

Visual design guidelines

  • The popover should be visually anchored to its trigger with an arrow or caret so the relationship between the trigger and the popover is immediately clear.
  • Allow sufficient padding around popover content to prevent text from touching the edges and to provide adequate touch targets for interactive elements inside.
  • The popover must appear at a z-index above all other page content and must not extend beyond the viewport — position it so it flips to the other side of the trigger if it would clip the edge.
  • Interactive elements inside the popover must have adequate touch targets (minimum 44×44 px) and clearly visible focus rings.

Dev guidelines for web

  • The trigger must be a <button> with aria-expanded="false" when closed and aria-expanded="true" when open. Use aria-controls pointing to the popover's id: <button aria-expanded="false" aria-controls="popover-options">Options</button>.
  • If the popover contains interactive content, give it role="dialog" with an appropriate aria-label or aria-labeledby. If it is purely informational (no interactive elements), role="tooltip" and aria-describedby on the trigger may be more appropriate — see the Tooltips page.
  • Move focus into the popover on open when it contains interactive content — to the container (tabindex="-1") or the first focusable element. Users who cannot see the popover must have focus inside it to know it exists.
  • Support Escape to close the popover. When Escape is pressed while focus is inside, close the popover and return focus to the trigger.
  • Return focus to the trigger on close regardless of the close mechanism — Escape, the close button, or clicking outside. Never leave focus inside a closed overlay.
  • If the popover does not use role="dialog", ensure that tabbing out of it also closes it, so it does not remain open and obscure content behind it.

Example code

<button
  type="button"
  id="filter-btn"
  aria-haspopup="dialog"
  aria-expanded="false"
  aria-controls="filter-popover"
>
  Filters
</button>

<div
  id="filter-popover"
  role="dialog"
  aria-labelledby="filter-heading"
  aria-modal="false"
  hidden
>
  <h3 id="filter-heading">Filter results</h3>
  <form>
    <label>
      <input type="checkbox" name="in-stock" />
      In stock only
    </label>
    <button type="submit">Apply filters</button>
  </form>
  <button
    type="button"
    aria-label="Close filters popover"
  >
    &#10005;
  </button>
</div>
