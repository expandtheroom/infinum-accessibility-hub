Bottom Sheets

Accessible bottom sheet patterns with focus management, keyboard support, and ARIA roles.

Design guidelines

UX guidelines

  • Include a visible handle indicator at the top of the sheet to communicate that it can be dragged. This is a widely understood mobile affordance.
  • Include a visible close button — do not rely solely on a drag gesture or tapping the backdrop, as these gestures are not accessible to all users.
  • Sheet height should be proportional to content and should not exceed 90vh — content within the sheet should scroll internally if needed, not the whole page.
  • Avoid making the sheet taller than 90vh — content that overflows should scroll within the sheet container.

Visual design guidelines

  • Use a dimmed backdrop behind the sheet to clearly communicate that the background content is temporarily inactive.
  • Entry animation should slide in from the bottom. Respect prefers-reduced-motion — provide a fade or instant-show alternative for users who have opted into reduced motion.
  • Interactive elements inside the sheet must have adequate touch targets (minimum 44×44 px) as bottom sheets are primarily a mobile pattern.

Dev guidelines for web

  • Give the sheet container role="dialog", aria-modal="true", and aria-labeledby pointing to the sheet's visible heading: <div role="dialog" aria-modal="true" aria-labelledby="sheet-title">.
  • Move focus into the sheet on open — to the sheet container (tabindex="-1"), the heading, or the first interactive element. Screen reader users will not know the sheet has appeared if focus stays on the trigger.
  • Trap focus within the sheet while it is open: Tab on the last focusable element cycles to the first; Shift+Tab from the first cycles to the last.
  • Support Escape to close — implement a keydown listener that responds to Escape regardless of which element inside currently has focus.
  • Return focus to the trigger on close — store a reference before opening and call trigger.focus() when the sheet closes, whether via the close button, Escape, or an action within the sheet.
  • Apply the inert attribute to all sibling containers while the sheet is open. Do not rely on aria-modal="true" alone — support across screen readers and browsers is inconsistent.
  • Drag-to-close is an enhancement only — always provide a visible <button> close control as the primary keyboard and screen reader dismissal mechanism.

Example code

<button
  type="button"
  aria-haspopup="dialog"
  onclick="document.getElementById('bottom-sheet').removeAttribute('hidden');
           document.getElementById('sheet-close').focus();"
>
  Open options
</button>

<div
  role="dialog"
  id="bottom-sheet"
  aria-modal="true"
  aria-labelledby="sheet-title"
  hidden
>
  <h2 id="sheet-title">Share</h2>
  <ul>
    <li><button type="button">Copy link</button></li>
    <li><button type="button">Send via email</button></li>
  </ul>
  <button
    type="button"
    id="sheet-close"
    aria-label="Close Share dialog"
    onclick="document.getElementById('bottom-sheet').hidden=true;"
  >
    Close
  </button>
</div>
