Tables

Accessible data table patterns with correct headers, captions, and cell associations.

Design guidelines

UX guidelines

  • Every table must have a visible caption or a heading immediately preceding it that clearly describes what the table contains.
  • Sortable column headers must show sort direction with an icon (e.g. an arrow) — not color alone — and the active sort column must be visually distinguishable from unsorted columns.
  • Tables that overflow horizontally on small screens need a visible scroll affordance (e.g. a fade shadow at the edge or a "scroll to see more" hint).
  • Never use a table for layout — use CSS Grid or Flexbox instead.

Visual design guidelines

  • Column headers must be visually distinct from data cells — use bold text, a contrasting background color, or both — so the header row is immediately identifiable.
  • Alternating row colors ("zebra striping") are optional but helpful for wide tables; ensure both row colors meet WCAG contrast requirements against cell text.
  • Cells must have sufficient padding — cramped cells are harder to read for users with low vision or cognitive impairments.

Dev guidelines for web

  • Provide a <caption> as the first child of <table> — this is the table's accessible name, announced before content, and listed in the screen reader's table navigation.
  • Use <th scope="col"> for column headers and <th scope="row"> for row headers so screen readers can associate data cells with the correct headers.
  • Complex tables with spanning headers need id attributes on <th> elements and headers="id1 id2" on the corresponding <td> cells — scope alone is insufficient for multi-level headers.
  • Avoid colspan/rowspan unless necessary; if used, always apply the id/headers technique to maintain programmatic associations.
  • Sortable columns: add aria-sort="ascending", aria-sort="descending", or aria-sort="none" on the active <th>; the sort trigger must be a <button> inside the <th> so it is keyboard-activatable with Enter or Space.
  • Scrollable tables on mobile: wrap in <div tabindex="0" role="region" aria-label="[table description]" style="overflow-x:auto"> so keyboard users can scroll the container.

Example code

<table>
  <caption>Q4 sales by region</caption>
  <thead>
    <tr>
      <th scope="col">Region</th>
      <th scope="col">Units sold</th>
      <th scope="col">Revenue</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Americas</th>
      <td>12,400</td>
      <td>$2.1M</td>
    </tr>
    <tr>
      <th scope="row">EMEA</th>
      <td>8,900</td>
      <td>$1.5M</td>
    </tr>
    <tr>
      <th scope="row">APAC</th>
      <td>3,800</td>
      <td>$0.6M</td>
    </tr>
  </tbody>
</table>
