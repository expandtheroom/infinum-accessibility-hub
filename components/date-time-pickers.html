Date & Time Pickers

Accessible date and time input patterns that work for keyboard users, screen readers, and people with cognitive disabilities.

Design guidelines

UX guidelines

  • Include the date format as a visible hint in the label or in hint text directly below the input — never in placeholder text alone. For example: "Date of birth (DD/MM/YYYY)". Placeholder text disappears when the user starts typing and is insufficient as the only format guide.
  • Error messages must be specific and actionable — write "Date must be today or in the future" rather than "Invalid date". Design error states to appear below the input with a color change plus an error icon, never color alone.
  • The calendar or time popup must be triggered by a clearly labeled button — typically a calendar icon button adjacent to the input. Annotate the button's accessible name in design files (e.g. "Choose date") so developers expose it correctly.
  • Avoid custom date pickers unless native inputs are insufficient — document the business reason in your design files. Custom calendar widgets carry significant accessibility risk and require extensive ARIA and keyboard implementation to match the native browser experience.

Visual design guidelines

  • The selected date must be visually distinct with accessible contrast — the selected day cell should meet a minimum 3:1 contrast ratio against adjacent unselected cells, not just against the page background. Avoid using only a subtle background fill that may not be distinguishable by users with low vision.
  • Today's date must be visually distinct from the selected date — use different styling (e.g. a border for today, a filled background for selected) so the two states are not confused by users with color blindness or low vision.
  • Disabled dates must look non-interactive — use reduced opacity, a strikethrough, or a visually muted color paired with a not-allowed cursor. Annotate in design files that disabled dates must also be communicated programmatically, not just styled.

Dev guidelines for web

  • Prefer <input type="date"> and <input type="time"> — browsers provide accessible, keyboard-operable date pickers at no implementation cost. Associate a visible <label> via for/id and link hint text via aria-describedby.
  • Link format hint text to the input: <input type="date" id="dob" aria-describedby="dob-hint" /> with <p id="dob-hint">Format: DD/MM/YYYY</p>. Screen readers announce the hint after the label when the field receives focus.
  • On validation failure, set aria-invalid="true" on the input and add the error message element's id to aria-describedby. Describe the failure specifically: "Date must be today or in the future".
  • If a custom calendar widget is required, implement the full ARIA grid pattern: role="grid" on the calendar, role="row" on each week row, and role="gridcell" on each day. Apply aria-selected="true" to the selected day and aria-disabled="true" to unavailable dates.
  • Keyboard navigation in a custom calendar grid: Arrow Left/Arrow Right move one day, Arrow Up/Arrow Down move one week, Page Up/Page Down move one month, Home/End jump to the first/last day of the week, Enter or Space select, Escape closes the dialog and returns focus to the trigger.
  • Use aria-label on each day cell to provide a full, unambiguous date — e.g. aria-label="14 March 2026" rather than just "14" — so screen reader users hear the complete date regardless of the visual calendar layout.
  • The calendar trigger button must have a descriptive label such as aria-label="Choose date". When the dialog opens, move focus into it. When it closes (via selection or Escape), return focus to the trigger button.

Example code

<div>
  <label for="appointment-date">
    Appointment date
    <span aria-hidden="true">*</span>
    <span class="visually-hidden">(required)</span>
  </label>
  <input
    type="date"
    id="appointment-date"
    name="appointment-date"
    required
    aria-describedby="date-hint"
    min="2025-01-01"
    max="2026-12-31"
  />
  <p id="date-hint">
    Select a date between January 2025 and December 2026.
  </p>
</div>

Select a date between January 2025 and December 2026.
