Progress Bars & Loaders / Spinners

Accessible progress bar and loading indicator patterns with screen reader announcements.

Design guidelines

UX guidelines

  • A determinate progress bar must show the current value both visually (the filled bar) and numerically — for example, "40% uploaded" — so users do not have to estimate from the graphic alone.
  • Always include a visible label that identifies what is loading (e.g. "Uploading file…" or "Processing payment"). A bare percentage without context is unhelpful.
  • Spinners must always have an associated text label nearby — either visible or visually hidden — that explains what is happening. A spinning graphic alone communicates nothing to users who cannot see it.
  • For indeterminate loaders, communicate the estimated wait time where possible (e.g. "This may take a minute") so users know whether to wait or return later.
  • Ensure the loading state does not obscure or replace a form submit button without updating its visible label — users must understand the button's current state.
  • Communicate loading completion or failure clearly — do not simply remove the spinner without replacing it with a result state or success message.
  • Time out loading animations after a reasonable period — around 30 seconds is a common threshold. When the timeout is reached, stop the animation and display a visible error or retry state (e.g. "Something went wrong. Try again."). Never leave a user watching a spinner indefinitely.

Visual design guidelines

  • Respect prefers-reduced-motion — replace spinning animations with a pulsing or static indicator for users who have opted into reduced motion, as continuous rotation can trigger vestibular disorders.

Dev guidelines for web

  • For determinate progress, use role="progressbar" with aria-valuenow, aria-valuemin="0", aria-valuemax="100", and aria-valuetext="40%". Always provide an accessible name via aria-label or aria-labeledby.
  • Use aria-valuetext for richer descriptions: e.g. aria-valuetext="Uploading, 40% complete" is more useful than a bare number.
  • For indeterminate loading, set aria-busy="true" on the loading region: <section aria-busy="true" aria-label="Loading results">. Set it to false when loading completes — do not use role="progressbar" with artificial values.
  • Mark the spinner SVG or icon as aria-hidden="true". Provide the accessible label via a visually-hidden sibling: <span class="visually-hidden">Loading search results, please wait.</span>
  • Announce completion via an aria-live="polite" region — inject a brief message such as "Search results loaded" when the spinner is removed and content appears.
  • If a spinner replaces a button label during an async action, update the button's accessible name: aria-label="Submitting, please wait". Restore it on completion.
  • Update aria-valuenow and aria-valuetext dynamically as progress changes, but avoid updating more than once per second to prevent excessive announcements.
  • Honour prefers-reduced-motion in CSS — wrap all spinner and progress bar animations in a motion-safe media query and provide a static or minimal fallback: 
    /* Default: animation on */
.spinner { animation: spin 0.8s linear infinite; }

@media (prefers-reduced-motion: reduce) {
  /* Remove continuous rotation; use opacity pulse instead */
  .spinner { animation: none; opacity: 0.6; }
  /* Or remove shimmer from skeleton/progress bar fills */
  .progress-fill { transition: none; }
}

Example code

<!-- Determinate progress bar -->
<div>
  <span id="upload-label">Uploading file</span>
  <div
    role="progressbar"
    aria-labelledby="upload-label"
    aria-valuenow="65"
    aria-valuemin="0"
    aria-valuemax="100"
    aria-valuetext="65% complete"
  >
    <div style="width: 65%"></div>
  </div>
  <span aria-hidden="true">65%</span>
</div>

<!-- Indeterminate spinner -->
<div
  role="status"
  aria-label="Loading, please wait"
>
  <span aria-hidden="true" class="spinner"></span>
</div>
Uploading file 65%
Processing…
