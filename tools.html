Tools

A guide to the tools that help teams test, audit, and experience the web through the lens of accessibility.

Screen Readers

Screen readers are assistive technology that converts on-screen content into synthesized speech or Braille output. Testing with real screen readers is the most reliable way to verify the accessibility of your product. The data below is drawn from the WebAIM Screen Reader Survey #10 (opens WebAIM).

Browser pairings

Screen readers and browsers must work together. Using a screen reader with a non-recommended browser can produce inconsistent or incorrect output. The most common combinations used by real screen reader users are:

Screen reader Browser % of users
JAWS Chrome 24.7%
NVDA Chrome 21.3%
JAWS Edge 11.4%
NVDA Firefox 10.0%
VoiceOver Safari 7.0%
NVDA Edge 5.0%

JAWS

Primary screen reader on Windows. JAWS (Job Access With Speech) by Freedom Scientific is the most widely used primary screen reader at 40.5% market share. Best paired with Chrome or Edge.

Quick start guide
  • Download JAWS (opens Freedom Scientific) — a 40-minute demo mode is available without a license
  • Start/stop: Launch from the desktop shortcut or Start menu. JAWS starts reading automatically. Press Insert+F4 to quit.
  • Reading: Insert+Down Arrow reads from the current position. Ctrl stops speech.
  • Navigation: Tab moves between focusable elements. H jumps to the next heading; Shift+H goes back. B jumps to buttons, F to form fields, L to lists.
  • Virtual cursor vs. forms mode: JAWS uses a virtual cursor for reading content. When a form field is focused it automatically switches to forms/application mode. Press Enter to manually enter forms mode; Numpad Plus or Esc to exit.
  • Links list: Insert+F7 opens a list of all links on the page — useful for spot-checking link text quality.
  • Headings list: Insert+F6 opens a list of all headings — useful for auditing heading structure.

NVDA

Free and open-source screen reader for Windows. NVDA (NonVisual Desktop Access) is used by 65.6% of screen reader users (many as a secondary reader) and is the go-to free option for accessibility testing. Best paired with Chrome or Firefox.

Quick start guide
  • Download NVDA (opens NV Access) — free, open-source
  • Start/stop: Launch from the desktop or Start menu. Press Insert+Q to quit (confirm in the dialog that appears).
  • Reading: Insert+Down Arrow reads from the current position. Ctrl stops speech.
  • Navigation: Tab between focusable elements. Single-letter quick keys work in browse mode: H for headings, B for buttons, F for form fields, L for lists, T for tables.
  • Browse vs. forms mode: NVDA uses browse mode for reading web content. It automatically switches to focus/forms mode when you interact with a form field. Press Insert+Space to toggle modes manually.
  • Elements list: Insert+F7 opens a filterable list of headings, links, and landmarks — excellent for auditing page structure.
  • Speech viewer: Tools → Speech Viewer shows all NVDA output as text — very helpful when testing without audio.

VoiceOver (macOS)

Built-in screen reader on macOS. VoiceOver is used by 43.9% of screen reader users and is the standard tool for testing on Apple platforms. Best paired with Safari.

Quick start guide
  • No download needed — built into macOS. Enable at System Settings → Accessibility → VoiceOver, or press Command+F5.
  • Start/stop: Command+F5 toggles VoiceOver on and off.
  • The VO modifier: Most VoiceOver commands use the VO key, which is Control+Option (abbreviated VO).
  • Reading: VO+A reads from the current position. Ctrl stops speech.
  • Navigation: Tab moves between interactive elements. VO+Right Arrow moves to the next item in the VoiceOver cursor. VO+U opens the rotor — a quick navigation menu for headings, links, landmarks, form controls, and tables.
  • Interacting with elements: VO+Shift+Down Arrow enters a group or widget; VO+Shift+Up Arrow exits it.
  • Quick Nav: Press Left Arrow+Right Arrow simultaneously to toggle Quick Nav, which lets you navigate with just the arrow keys without holding VO.

VoiceOver (iOS)

Built-in screen reader on iPhone and iPad. VoiceOver on iOS is the dominant mobile screen reader — used by 70.6% of mobile screen reader users. Best paired with Safari.

Quick start guide
  • No download needed — built into iOS. Enable at Settings → Accessibility → VoiceOver, or ask Siri "Turn on VoiceOver". Set up the Accessibility Shortcut (triple-click the side/home button) for quick toggling.
  • Reading: Tap any element to hear it. Swipe right to move to the next element; swipe left for the previous.
  • Activating elements: Double-tap to activate the focused element (equivalent to a tap).
  • Scrolling: Use a three-finger swipe up or down to scroll.
  • The rotor: Rotate two fingers on screen like a dial to open the rotor, which lets you change navigation mode (headings, links, words, characters, etc.). Once a mode is set, swipe up/down to navigate by that unit.
  • Escape: Two-finger scrub (move two fingers back and forth in a Z shape) to go back or dismiss an overlay — equivalent to pressing Escape or a back button.

TalkBack (Android)

Built-in screen reader on Android. TalkBack is used by 34.7% of mobile screen reader users and is the primary tool for testing accessibility on Android devices. Best paired with Chrome.

Quick start guide
  • No download needed — built into Android. Enable at Settings → Accessibility → TalkBack. The exact path varies by device manufacturer. Set up the accessibility shortcut (hold both volume keys) for quick toggling.
  • Reading: Tap any element to hear it. Swipe right to move to the next element; swipe left for the previous.
  • Activating elements: Double-tap to activate the focused element.
  • Scrolling: Use a two-finger swipe up or down to scroll.
  • Reading controls: Swipe up then right (or down then right) to open reading controls — TalkBack's equivalent of VoiceOver's rotor. Use it to navigate by headings, links, controls, or paragraphs.
  • Global context menu: Swipe down then right to open the global context menu, which provides quick access to navigation and reading options.

Narrator

Built-in screen reader on Windows. Narrator is included with all Windows installations and requires no setup. While it has a low primary usage rate (0.7%), it is commonly used as a secondary reader (37.3%) and is useful for quick spot-checks on machines where JAWS or NVDA are not installed. Best paired with Edge.

Quick start guide
  • No download needed — built into Windows 10 and 11. Press Windows+Ctrl+Enter to toggle Narrator on and off.
  • Reading: Narrator key+M starts reading from the current position. The Narrator key is Caps Lock or Insert (configurable).
  • Navigation: Tab between interactive elements. Narrator key+F5 lists headings; Narrator key+F6 lists landmarks; Narrator key+F7 lists links.
  • Scan mode: Narrator key+Space toggles scan mode, which allows navigation with arrow keys and single-letter shortcuts similar to NVDA's browse mode.

Auditing Tools

Automated tools can catch a significant portion of accessibility issues quickly, but they cannot catch everything — estimates suggest automated tools find around 30–40% of all WCAG failures. Always combine automated scanning with manual keyboard testing and screen reader testing.

axe DevTools

axe DevTools (opens Deque) by Deque is the industry-standard accessibility testing engine, used as the foundation for many other tools including Lighthouse. It is available as a browser extension for Chrome and Firefox, and as a library that can be integrated into automated test suites.

Lighthouse

Lighthouse (opens Chrome Developers) is an open-source, automated auditing tool built into Chrome DevTools. It runs audits for performance, SEO, and accessibility, and is useful for a quick health check during development.

WAVE

WAVE (opens WebAIM) (Web Accessibility Evaluation Tool) by WebAIM provides visual feedback about the accessibility of web content by injecting icons and indicators directly into the page.

Keyboard testing

Keyboard testing requires no tools — just unplug or disable your mouse and attempt to use the page using only the keyboard. It is one of the most effective manual tests you can run, and it surfaces issues that no automated tool can catch.

  • Tab — move focus forward through interactive elements (links, buttons, inputs)
  • Shift+Tab — move focus backward
  • Enter — activate a link or button
  • Space — activate a button, checkbox, or select; scroll the page
  • Arrow keys — navigate within a widget (radio group, listbox, tab panel, slider)
  • Escape — dismiss a modal, tooltip, dropdown, or other overlay

Things to check during keyboard testing:

  • Focus visibility — can you always see which element has focus? The focus indicator must be clearly visible at all times.
  • Focus order — does focus move in a logical, meaningful sequence that matches the visual layout?
  • No focus traps — except in modals (where trapping is intentional), can you always Tab away from any component?
  • All interactive elements reachable — every button, link, and form field must be reachable by keyboard.
  • No keyboard-only bugs — do dropdowns, date pickers, and custom widgets open, operate, and close correctly without a mouse?

Browser Extensions

HeadingsMap

HeadingsMap (opens Chrome Web Store) generates a visual map of the heading structure of any page in a sidebar panel. It is one of the fastest ways to audit heading hierarchy without a screen reader.

  • Shows all headings (h1–h6) in a nested tree, making skipped levels and incorrect nesting immediately obvious.
  • Clicking a heading in the panel scrolls to and highlights that heading on the page.
  • Also displays ARIA landmark regions and the document outline, useful for verifying overall page structure.
  • Available for Chrome (opens Chrome Web Store) and Firefox (opens Firefox Add-ons).
